# 🛡 여의도 맛집 가이드 - 관리자 메뉴얼

본 서비스는 Google Sheets를 백엔드로 활용하는 정적 웹 애플리케이션입니다. 사용자 제안 데이터를 관리하고 반영하는 방법을 설명합니다.

## 1. 데이터 저장소 구조 (Google Sheets)
사용자 제안은 Google Apps Script를 통해 구글 시트의 아래 시트들에 저장됩니다.

- **Pending (대기열)**: 사용자가 웹에서 제안한 데이터가 즉시 쌓이는 곳입니다.
- **Approved (승인됨)**: 관리자가 검토 후 웹사이트에 노출하기로 결정한 데이터들을 모아두는 곳입니다.

## 2. 데이터 승인 프로세스
1. 구글 시트의 `Pending` 탭을 열어 새로 들어온 제안을 확인합니다.
2. 유효한 제안인 경우, 해당 데이터를 `Approved` 탭으로 이동시킵니다.
   - *팁: Google Apps Script를 수정하여 승인 버튼 하나로 이동하게 만들 수도 있습니다.*

## 3. 웹사이트 업데이트 (커맨드 라인)
승인된 데이터를 실제 소스코드(`data.js`)에 반영하기 위해 다음 과정을 수행합니다.

1. **데이터 가져오기**:
   ```bash
   node update_data.js
   ```
   - 이 명령어는 구글 시트 API를 호출하여 `Approved` 시트의 내용으로 `data.js`를 자동 갱신합니다.

2. **반영 확인**:
   - `index.html`을 열어 데이터가 정상적으로 추가되었는지 확인합니다.

3. **배포**:
   - 변경된 `data.js`를 GitHub에 푸시합니다.
   - Vercel이 푸시를 감지하고 자동으로 정적 페이지를 재빌드하여 배포합니다.

## 4. 주의사항
- `update_data.js`를 실행하기 전, 파일 내의 `GAS_READ_URL`이 실제 배포된 구글 앱스 스크립트의 URL인지 반드시 확인하세요.
- `data.js`는 정적 파일이므로, 구글 시트에서 승인만 한다고 해서 즉시 웹에 나타나지 않습니다. 반드시 **스크립트 실행(`node update_data.js`) + 커밋** 과정이 필요합니다.
